# Internationalization (i18n) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `react-i18next`-based internationalization to the OpenChore frontend with a language selector, browser auto-detection, English fallback, and a complete extracted English translation file.

**Architecture:** A single i18next instance (`web/src/i18n/`) is initialized once in `main.tsx`. Language is auto-detected from the browser, selectable via a UI dropdown, and persisted in `localStorage` (`openchore_lang`) — no backend changes. All static UI strings are replaced with `t('area.key')` lookups against `locales/en/translation.json`. A central `LANGUAGES` registry drives the selector and supported-language config so adding a language is one line + one JSON file.

**Tech Stack:** React 18, TypeScript, Vite, `i18next`, `react-i18next`, `i18next-browser-languagedetector`, Vitest.

**Spec:** `docs/superpowers/specs/2026-06-10-i18n-design.md`

- [ ] **Step 4: Create the i18next init module**

Create `web/src/i18n/index.ts`:

```ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';
import { SUPPORTED_CODES, FALLBACK_CODE } from './languages';
import en from './locales/en/translation.json';

// resources is keyed by language code; add new languages alongside their
// LANGUAGES entry. Each maps to a single `translation` namespace.
const resources = {
  en: { translation: en },
} as const;

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources,
    supportedLngs: SUPPORTED_CODES,
    fallbackLng: FALLBACK_CODE,
    nonExplicitSupportedLngs: true, // 'en-US' -> 'en'
    returnNull: false,
    interpolation: { escapeValue: false }, // React already escapes
    detection: {
      order: ['localStorage', 'navigator'],
      lookupLocalStorage: 'openchore_lang',
      caches: ['localStorage'],
    },
  });

export default i18n;
```

```tsx
import React from 'react';
import { useTranslation } from 'react-i18next';
import { LANGUAGES } from '../../i18n/languages';
import styles from './LanguageSelector.module.css';

interface Props {
  className?: string;
}

export const LanguageSelector: React.FC<Props> = ({ className }) => {
  const { i18n, t } = useTranslation();

  return (
    <select
      className={[styles.select, className].filter(Boolean).join(' ')}
      value={i18n.resolvedLanguage}
      onChange={(e) => i18n.changeLanguage(e.target.value)}
      aria-label={t('common.language', 'Language')}
    >
      {LANGUAGES.map((lng) => (
        <option key={lng.code} value={lng.code}>
          {lng.label}
        </option>
      ))}
    </select>
  );
};

export default LanguageSelector;
```

## Extraction recipe (Tasks 6–N)

Each extraction task converts one file's hardcoded UI strings into `t('area.key')`
lookups. **Follow this exact recipe per file:**

1. Add the hook at the top of the component body:
   ```tsx
   import { useTranslation } from 'react-i18next';
   // ...inside the component:
   const { t } = useTranslation();
   ```
2. For every user-facing static string — JSX text, `placeholder`, `aria-label`,
   `title`, `alt` that is descriptive, button labels, toast/error messages —
   add a key under the file's area namespace in `en/translation.json` and replace
   the literal with `t('area.key')`.
3. For strings with variables, use interpolation:
   `t('dashboard.greeting', { name: user.name })` ↔ `"greeting": "Hi, {{name}}"`.
4. For count-dependent strings, use plurals:
   `t('dashboard.points', { count: n })` ↔ `points_one` / `points_other`.
5. **Do NOT translate** backend/API/config data (chore titles, user names, seeded
   categories, icon names) or non-descriptive `alt={user.name}`.
6. After each file: `npx tsc --noEmit` passes, then commit.

**Worked example — `ProfileSelection.tsx`** (covered in Task 6 below) shows the
full before/after. Apply the identical recipe to every file in Tasks 7–N.
